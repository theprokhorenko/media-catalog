# 🎬 Media Catalog

Бэкенд-сервис каталога фильмов и сериалов с рекомендательной системой. Написан на FastAPI + SQLAlchemy, с простым веб-интерфейсом поверх REST API.

Проект демонстрирует полный цикл: проектирование реляционной БД, CRUD-API с валидацией, наполнение из реального датасета (~8800 тайтлов) и контентную рекомендательную систему.

## Возможности

- **CRUD** для тайтлов и жанров (создание, чтение, обновление, удаление)
- **Связь many-to-many** между тайтлами и жанрами через ассоциативную таблицу
- **Поиск** по названию и **фильтрация** по типу и жанру
- **Пагинация** выдачи
- **Рекомендательная система** — подбор похожих тайтлов по пересечению жанров и совпадению режиссёра
- **Веб-интерфейс** — страница с карточками, поиском, фильтрами и кнопкой «Похожее»
- **Автодокументация API** (Swagger UI) из коробки
- Наполнение базы из открытого датасета Netflix

## Стек

- **Python 3.12**
- **FastAPI** — веб-фреймворк, REST API
- **SQLAlchemy** — ORM
- **Pydantic** — валидация данных и схемы
- **SQLite** — база данных
- **pandas** — обработка датасета при заливке
- **Uvicorn** — ASGI-сервер
- **HTML / CSS / JavaScript** — веб-интерфейс (fetch к API)

## Установка и запуск

### 1. Клонировать репозиторий

```
git clone https://github.com/theprokhorenko/media-catalog.git
cd media-catalog
```

### 2. Создать и активировать виртуальное окружение

```
python -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate
```

### 3. Установить зависимости

```
pip install -r requirements.txt
```

### 4. Создать таблицы

```
python init_db.py
```

### 5. Скачать датасет и наполнить базу

Скачайте датасет **Netflix Movies and TV Shows** с Kaggle:
https://www.kaggle.com/datasets/shivamb/netflix-shows

Положите файл `netflix_titles.csv` в корень проекта и запустите:

```
python load_data.py
```

Скрипт загрузит ~8800 тайтлов с жанрами. Если файл не найден — выведет подсказку со ссылкой.

### 6. Запустить сервер

```
uvicorn main:app --reload
```

- Веб-интерфейс: http://127.0.0.1:8000/app
- Документация API (Swagger): http://127.0.0.1:8000/docs

## Основные эндпоинты

| Метод  | Путь                                | Описание                          |
|--------|-------------------------------------|-----------------------------------|
| GET    | `/titles`                           | Список тайтлов (поиск, фильтры, пагинация) |
| GET    | `/titles/{id}`                      | Один тайтл по id                  |
| POST   | `/titles`                           | Создать тайтл                     |
| GET    | `/titles/{id}/recommendations`      | Рекомендации похожих тайтлов      |
| GET    | `/genres`                           | Список жанров                     |
| POST   | `/genres`                           | Создать жанр                      |
| PUT    | `/genres/{id}`                      | Обновить жанр                     |
| DELETE | `/genres/{id}`                      | Удалить жанр                      |

## Архитектурные решения

**Модель данных.** Все виды контента хранятся в единой таблице `titles` с полем-дискриминатором `media_type` — отдельные таблицы под каждый вид не нужны, так как отличаются они только значением одного атрибута. Жанры вынесены в отдельную таблицу и связаны с тайтлами отношением many-to-many через ассоциативную таблицу.

**Разделение слоёв.** ORM-модели (хранение) и Pydantic-схемы (контракт API) разделены: схемы на ввод и на вывод отличаются, что позволяет валидировать входные данные и контролировать формат ответа независимо от структуры БД.

**Рекомендательная система.** Контентный подход: для заданного тайтла вычисляется score с каждым другим тайтлом как размер пересечения множеств жанров плюс бонус за совпадение режиссёра; возвращается топ по score. Реализация — простой перебор (подходит для текущего объёма данных).

## Дальнейшее развитие

- **Коллаборативная фильтрация** — рекомендации на основе оценок пользователей («похожий вкус»); требует модели пользователей, аутентификации и накопленных оценок.
- **Улучшение рекомендаций по режиссёру** — в датасете режиссёры могут быть списком; сейчас сравнивается полная строка, в развитии — разбивать и сравнивать множества (как с жанрами).
- **Ранжирование поиска по релевантности** — сейчас результаты сортируются по порядку загрузки в БД; точные совпадения и совпадения в начале названия стоило бы поднимать выше, сортируя по степени соответствия запросу.
- **Оптимизация рекомендаций** — при росте объёма данных заменить перебор на предпосчёт similarity или векторный поиск.
- **Миграции БД** (Alembic) — вместо пересоздания базы при изменении схемы.
- **Полноценная пагинация** — отдавать общее число записей для корректного отображения числа страниц.
